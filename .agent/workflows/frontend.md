## Coding Conventions

### Components

1. **Use functional components** with TypeScript and make the component as dynamic as possible, reusable, customizable, and type-safe.
2. **Use server component first** when possible. Avoid using the `use client` directive when there is no frontend interaction in the component.
3. **Use default exports** for page components and named exports for reusable components
4. **Use `@/` path alias** for imports (e.g., `@/components/ui/button`)

```tsx
"use client";
import { Button } from "@/components/ui/button";

export default function MyComponent() {
  return <Button>Click me</Button>;
}
```
5. **Page components such as `layout.tsx` and `page.tsx` should be a server component** when possible to ensure the contents are SEO-friendly.

### TypeScript

1. **Use type-only imports** for types: `import type { MyType } from "./types"`
2. **Prefer `interface`** for component props
3. **Use `as const`** for literal arrays/objects
4. **Avoid `any`** - use `unknown` if type is truly unknown
5. **Use 2 space based indentation** to make the code more concise.

### Styling

1. **Use Tailwind CSS utility classes** for styling
2. **Use `cn()` helper** from `@/lib/utils` for conditional classes:
   ```tsx
   import { cn } from "@/lib/utils";
   
   <div className={cn("base-class", isActive && "active-class")} />
   ```
3. **Use CSS variables** for theme colors (defined in `index.css`):
   - `bg-background`, `text-foreground` (base colors)
   - `bg-primary`, `text-primary-foreground` (primary colors)
   - `bg-muted`, `text-muted-foreground` (muted colors)
   - `bg-card`, `border-border` (card styles)
4. **Dark mode** is handled via `.dark` class and CSS variables
5. **Avoid using Tailwind arbitrary classes** when possible
6. **Avoid editing the `index.css`**
7. **Avoid gradient colors and rounded corners** to make the style consistent being sharpy and solid.

## shadcn/ui Components

- Located in `@/components/ui/`
- **Do not edit** these files directly (they're auto-generated by shadcn CLI)
- Add new components via: `npx shadcn@latest add <component-name>`
- Configuration in `components.json`

## API Calls

Use the Axios instance from `@/lib/api/`:
```tsx
import { api } from "@/lib/api";

// in a React Query hook or component
const { data } = useQuery({
  queryKey: ["items"],
  queryFn: () => api.get("/items").then(res => res.data),
});
```

## Best Practices

1. **Colocation**: Keep component-specific logic close to the component
2. **Composition**: Prefer composition over prop drilling
3. **Server Components**: Use server components by default, add `"use client"` only when needed
4. **Loading States**: Use `<Skeleton />` from shadcn/ui for loading states
5. **Error Handling**: Use error boundaries and toast notifications (Sonner)
6. **Accessibility**: All interactive elements must be keyboard accessible
